# 队员调研内容补齐计划

## 目标

在保留生产现有数据的前提下，按“调研问题 -> 证据 -> 调研报告”补齐未完成成员，使每个已有问题都有验证状态，并让页面按真实状态计算到 100% 验证。推断内容必须标注资料来源和证据等级，不冒充现场事实。

## 阶段

- [complete] 1. 备份生产状态并盘点成员/站点缺口
- [complete] 2. 解析插入内容、现场留痕、知识库和现有高质量样本
- [complete] 3. 生成增量问题验证、证据记录和四核报告
- [complete] 4. 预演校验并写入 164 生产环境
- [complete] 5. 验证刷新持久化、页面比例和报告可读性

## 数据原则

- 不删除、不重置、不覆盖其他成员已有内容。
- 已有现场证据优先；桌面资料只能支持推断性结论。
- 没有充分证据时使用 partial/refuted 并写明限制，不编造采访对象、原话或精确数字。
- 每次生产写入前重新读取最新状态，按稳定 ID 增量合并。
- 写入前后分别备份 `app-state.json` 和上传目录。
- 删除旧第九组宪章，替换为第八组红八宝宪章，不保留旧成员信息。
