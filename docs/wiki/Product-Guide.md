# 产品使用指南

## 1. 队员端

进入 `/app`，选择成员身份、日期、线路和站点。队员可以维护预设问题、填写验证状态，并上传图片、录音和文字留痕。

验证结论应区分：

- 成立：已有明确现场依据。
- 部分成立：只验证了假设的一部分。
- 推翻：现场信息与原假设相反。
- 待验证：缺少足够证据，不应为了提高比例强行判断。

## 2. 留痕与知识

`/traces` 用于按成员、日期、站点和类型筛选现场证据；`/knowledge` 用于挂载团队资料并建立可检索片段。

## 3. 红八宝 Agent

`/agent` 优先检索本组资料。输出中的“本组事实”可以进入报告；“通识补充”和“待验证”必须继续寻找证据，不能直接当作企业结论。

## 4. 痛点与方案

协同 Hub 将问题和证据收敛为痛点，再为痛点建立最小试验方案。方案至少应包含目标用户、试验动作、衡量指标、风险、人工兜底和停止条件。

## 5. 报告与评审

`/dashboard` 按现状扫描、人群共情、痛点诊断、分析对策四核结构组织报告。管理员可上传 DOCX 定稿，教师通过 `/review` 查看代表站点、证据和下载入口。

## 6. 路演

`/screen/roadshow` 提供 26 幕电脑演示，支持播放、暂停、上一幕、下一幕、从头播放、空格键和方向键。演示读取真实数据，不会修改业务状态。
