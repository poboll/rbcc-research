# 安全与管理员访问

## 管理员密码是什么

项目没有硬编码或公开的默认管理员密码。管理端使用服务端环境变量 `ADMIN_TOKEN` 作为访问令牌：

```bash
ADMIN_TOKEN="请设置一个强随机令牌" npm start
```

打开 `/admin` 后输入同一个值。浏览器会把令牌保存在当前浏览器的本地存储中，并在管理请求的 `x-admin-token` 请求头中发送；服务端才是最终校验方。

本地开发未设置令牌时，服务端会允许管理操作以方便调试；`NODE_ENV=production` 或 Vercel 环境未设置令牌时，管理接口会返回 503。生产环境必须显式配置。

## 生成与保存令牌

可用系统密码生成器创建至少 32 字节的随机值。令牌只应保存在：

- Linux 的 root 可读环境文件，例如 `/etc/rbcc-research.env`；
- Vercel Project Settings 的加密环境变量；
- 团队认可的密码管理器。

不要将令牌写入 README、Wiki、`.env.example` 的实际值、截图、群聊或 Git 历史。

## 轮换

1. 在服务端设置新的 `ADMIN_TOKEN`。
2. 重启或重新部署服务。
3. 验证旧令牌返回 401、新令牌能够读取管理状态。
4. 通知授权管理员更新，不发送到公开群聊。

## 其他密钥

- `DEEPSEEK_API_KEY`：只允许服务端读取。
- `BLOB_READ_WRITE_TOKEN`：只用于 Vercel Blob，不能暴露到 `VITE_*` 变量。
- `SUPABASE_SERVICE_ROLE_KEY`：如启用 Supabase，只能在服务端使用。

## 数据原则

- 不把生产状态、现场附件或真实密钥提交到仓库。
- 管理导出和备份按可能包含个人信息处理。
- AI 生成内容必须标注来源边界，不将模型推断包装成现场证据。
- 发布与删除操作需要服务端授权；仅隐藏前端按钮不等于安全。
