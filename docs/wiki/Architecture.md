# 系统架构

## 运行结构

```text
React/Vite SPA
  |-- 队员端与现场上传
  |-- 协同 Hub 与拓扑
  |-- 知识中心与红八宝
  |-- 四核报告与评审
  `-- 26 幕路演
          |
          v
Node.js ESM API
  |-- JSON 状态存储: data/app-state.json
  |-- 媒体文件: data/uploads/
  |-- DeepSeek 服务端调用
  `-- DOCX 生成与下载
```

## 数据链

成员、站点和路线提供业务上下文；问题关联验证结论与证据；痛点聚合多个问题和证据；方案关联痛点；报告引用对应记录 ID。首页、协同 Hub、评审和路演只读取同一份生产状态。

## 存储边界

当前生产使用单节点 JSON 与本地 uploads，发布时必须排除 `data/`。单实例写队列降低并发覆盖风险，但 JSON 不是事务数据库。多实例部署前应迁移到 PostgreSQL，并使用独立对象存储保存媒体。

## 安全边界

- Agent 密钥只存在于服务端环境。
- 管理写接口校验 `x-admin-token`。
- 上传校验文件类型、签名、大小和文件名。
- 报告生成以团队资料为优先来源，并区分现场事实与模型通识。
