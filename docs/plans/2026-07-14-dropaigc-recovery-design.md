# dropaigc.com 完整恢复设计

## 恢复目标

以线上发布包作为视觉与交互的权威基线，保持所有公开页面、中文文案、导航、数据密度和响应式行为一致。同时重建可维护的服务端源码，使协作、上传、知识库、红小九 Agent、报告编辑、万字报告生成和 Word 导出能够在本地独立运行。

## 方案选择

采用“精确前端归档 + 可维护后端重建”的两层方案。公开构建不含 source map，无法数学意义上还原原始 TypeScript 文件、变量名和私有数据库实现；直接重写全部 React 页面会在短期内引入大量视觉偏差。保留浏览器实际执行的 Next.js 构建产物，能够保证一比一表现。服务端则按客户端 bundle 暴露的 REST 契约模块化重建，并把线上公开数据同步为本地种子。

## 数据流

队员上传文本、图片或录音后，服务端写入本地上传目录和统一状态文件。问题、现场证据、知识库与方案通过成员、站点、分组标识关联。红小九检索当前站点相关材料，构造受证据约束的 DeepSeek 请求，并返回引用。报告工作台把材料归并到现状扫描、人群共情、痛点诊断、分析对策四个板块；用户可保存草稿、请求 AI 提炼、生成完整报告并导出 DOCX。教师评审页只读取已落盘版本，避免评审期间内容漂移。

## 安全边界

DeepSeek 密钥只从服务端环境变量 `DEEPSEEK_API_KEY` 读取，绝不进入前端 bundle、JSON 数据或版本库。上传文件限制类型与大小，文件名在服务端重新生成。Agent 提示词明确区分事实、推断和待验证项，缺少证据时不得编造。

## 验收

逐路由比较线上与本地桌面截图，并在移动视口检查导航、正文和控件不溢出。核心流程覆盖新增知识、上传留痕、Agent 问答、报告草稿保存、报告生成和 Word 下载。接口测试同时验证无密钥时的可解释降级与配置密钥后的真实模型路径。
